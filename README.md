## Über TV-Rank

**TV-Rank** ist eine umfassende Tracking-App für **Serien, Filme, Anime, Manga, Manhwa & Manhua**. Behalte den Überblick über deinen Fortschritt bis auf Episoden- und Kapitelebene, bewerte deine Favoriten, entdecke Neues per KI, verwalte deine Streaming-Abos und miss dich mit deinen Freunden. Als **installierbare PWA** und **Windows-Desktop-App** verfügbar – mobile-first, offline-fähig und komplett individualisierbar.

> 🌐 Live unter **[tv-rank.de](https://tv-rank.de)**

---

## Home – dein Cockpit

Alles Wichtige auf einen Blick: personalisierte Begrüßung, Live-Statistiken, Schnellzugriffe, ein Countdown zur nächsten Staffel, dein „Weiterschauen"-Stapel und smarte Benachrichtigungen (neue Staffeln, Anbieter-Wechsel, inaktive Serien). Das Layout lässt sich frei anordnen.

## 🚀 Installation & Entwicklung

> **Hinweis:** Dies ist das Frontend. Es benötigt eine Firebase-Realtime-Database und die statische Katalog-API des Backends.

**1. Repository klonen & Abhängigkeiten installieren**

```bash
git clone https://github.com/Floatyy1998/Serien-Ranking.git
cd Serien-Ranking
npm install
```

**2. `.env` im Projekt-Stammverzeichnis anlegen**

```env
VITE_FIREBASE_API_KEY=<dein-api-key>
VITE_FIREBASE_AUTH_DOMAIN=<dein-projekt>.firebaseapp.com
VITE_FIREBASE_DATABASE_URL=https://<dein-projekt>.firebaseio.com
VITE_FIREBASE_PROJECT_ID=<deine-projekt-id>
VITE_FIREBASE_STORAGE_BUCKET=<dein-projekt>.appspot.com
VITE_FIREBASE_MESSAGING_SENDER_ID=<sender-id>
VITE_FIREBASE_APP_ID=<app-id>
VITE_FIREBASE_MEASUREMENT_ID=<measurement-id>
VITE_API_TMDB=<dein-tmdb-api-key>
VITE_API_OMD=<dein-omdb-api-key>
VITE_BACKEND_API_URL=<url-der-backend-api>
```

**3. Entwicklungsserver starten**

```bash
npm run dev          # Web-Dev-Server (http://localhost:5173)
npm run dev:electron # Web + Electron-Desktop-Fenster
```

### Nützliche Skripte

| Befehl | Beschreibung |
|---|---|
| `npm run dev` | Vite-Entwicklungsserver |
| `npm run build:local` | Produktions-Build (ohne Deploy) |
| `npm run build:electron` | Windows-Desktop-App bauen |
| `npm run lint` / `lint:fix` | ESLint |
| `npm run format` / `format:check` | Prettier |
| `npm run type-check` | TypeScript-Prüfung |
| `npm run preview` | Vorschau des Produktions-Builds |

---

## 📦 Deployment

Ein **Push auf `master`** löst über GitHub Actions die Pipeline aus: `format:check` → `lint` → `type-check` → `vite build` → **Firebase Hosting Deploy** (Produktion). Prettier-, ESLint- oder TypeScript-Fehler blockieren das Deployment.
